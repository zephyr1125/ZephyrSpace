# SpaceX

## 公司简介

SpaceX 成立于 2002 年，是当前全球最重要的商业航天公司之一。它的核心不是单一火箭业务，而是把可重复使用发射、载人/货运飞船、卫星部署和卫星互联网运营整合成一个统一体系。

## 产业链位置

- 上游需求：发动机、结构材料、电子元器件、卫星载荷、地面终端
- 核心位置：[[02-主题/商业发射]]、[[02-主题/低轨卫星互联网]]
- 下游延伸：[[02-主题/卫星通信]]、政府与商业航天运输服务、消费级与企业级连接服务
- 国家维度：[[03-国家地区/美国商业航天]]

## 核心业务

- 以 Falcon 9 / Falcon Heavy 为核心的商业发射服务
- 以 Dragon 为核心的货运与载人航天服务
- 以 Starship 为核心的下一代重型、可复用运输体系研发
- 以 Starlink 为核心的低轨卫星互联网网络建设与运营

## 关键产品/服务

- Falcon 9：现阶段商业发射主力，也是 SpaceX 可重复使用能力的现金流基础
- Falcon Heavy：重型发射能力补位产品
- Dragon / Crew Dragon：对接 NASA 与商业载人任务的重要平台
- Starship：面向大运力、低成本、深空运输和大规模卫星部署的下一代平台
- Starlink：覆盖消费、企业、海事、航空等场景的低轨宽带网络

## 与 SpaceX 的关系

- 它本身是这个知识网络的锚点公司。
- [[01-公司/Rocket Lab]] 是美国发射与综合航天平台路径的可比样本，但规模、客户结构和一体化程度明显不同。
- [[01-公司/Amazon Project Kuiper]] 在低轨卫星互联网上与 SpaceX 正面竞争。
- [[01-公司/Viasat]] 则代表传统卫星通信公司在 Starlink 冲击下的适应路径。
- [[01-公司/蓝箭航天]] 与 [[01-公司/银河航天]] 分别构成中国在火箭和卫星互联网方向上的对照观察对象。

## 研究观察

- SpaceX 的真正护城河不只是火箭，而是“发射能力 + 制造能力 + 运营网络 + 终端产品”的系统能力。
- Starlink 让 SpaceX 从典型的项目制航天公司，部分转向具备运营商特征的平台型公司。
- Starship 的意义不止是单次任务能力提升，更在于是否能继续压低卫星部署和深空任务的边际成本。

## 个人观察

- 关注点：Starship 的工程节奏会不会真正传导到成本曲线，而不只是形成新的资本叙事。
- 关注点：Starlink 的用户规模、网络容量和 ARPU 能否长期支撑 SpaceX 更高估值预期。

## 待验证问题

- Starlink 的长期利润结构更接近电信运营商，还是更接近基础设施平台？
- Starship 成熟后，SpaceX 的优势会继续扩大，还是会给其他卫星/发射公司打开新机会？
